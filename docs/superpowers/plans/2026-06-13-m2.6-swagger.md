# M2.6 — Swagger(OpenAPI) 연동 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **문서 규칙:** 예시 코드는 구현 중에만 참고용으로 유지하고, PR 작성 직전에 제거했다(산문·시그니처·명령만 남김). 실제 구현은 각 파일과 커밋 이력 참조.

**Goal:** `@nestjs/swagger`로 API 문서/UI(`/docs`, `/docs-json`)를 자동 생성하고, 기존 3컨트롤러(auth/property/board)를 데코레이터로 문서화하며, **enum 필드의 허용값을 명명 스키마로 노출**한다. 신규 API Swagger 필수 규칙을 정립한다.

**Architecture:** Swagger 셋업을 `setupSwagger(app)` 함수로 추출해 `main.ts`(부팅)와 e2e가 공유 → `/docs-json`을 e2e로 검증 가능. enum은 `@ApiProperty({ enum, enumName })`로, M2.5 에러 봉투는 `ErrorResponseDto`로 문서화. 응답 DTO는 만들지 않고 `@ApiResponse` 설명/예시로 처리.

**Tech Stack:** NestJS 11, `@nestjs/swagger`, TypeScript, Jest. (스펙: `docs/superpowers/specs/2026-06-13-swagger-integration-design.md`)

**선행:** M2.5 머지(dev). 브랜치 `feat/m2.6-swagger`(dev에서 분기). 인프라(docker) 기동.

## Task 3: Auth 문서화 (@ApiTags/@ApiOperation/@ApiResponse/@ApiProperty)

**Files:** Modify `auth.controller.ts`, `dto/sign-up.dto.ts`, `dto/login.dto.ts`.

- [ ] **Step 1: 요청 DTO `@ApiProperty`**

- `sign-up.dto.ts`: 각 필드에 `@ApiProperty({ example, description })` (email/name/password). 기존 class-validator 데코레이터는 유지.
- `login.dto.ts`: email/password에 `@ApiProperty`.

- [ ] **Step 2: 컨트롤러 데코레이터**

`auth.controller.ts` 상단 `@ApiTags('auth')`. 라우트별:
- `POST /auth/signup`: `@ApiOperation({ summary: '회원가입' })`, `@ApiResponse({ status: 201, description: '생성된 유저' })`, `@ApiResponse({ status: 400, type: ErrorResponseDto })`, `@ApiResponse({ status: 409, type: ErrorResponseDto, description: '이메일 중복' })`.
- `POST /auth/login`: `@ApiOperation({ summary: '로그인(JWT 발급)' })`, `@ApiResponse({ status: 201, description: 'accessToken' })`, `@ApiResponse({ status: 401, type: ErrorResponseDto })`.
- `GET /auth/me`: `@ApiBearerAuth(SWAGGER_BEARER_AUTH)`, `@ApiOperation({ summary: '내 정보' })`, `@ApiResponse({ status: 200 })`, `@ApiResponse({ status: 401, type: ErrorResponseDto })`. 응답에 `role`(Role enum) 포함을 설명에 명시.

import: `@nestjs/swagger`의 `ApiTags/ApiOperation/ApiResponse/ApiBearerAuth`, `ErrorResponseDto`, `SWAGGER_BEARER_AUTH`.

## Task 5: Board 문서화 (+ enum 허용값 표기 핵심)

**Files:** Modify `board.controller.ts`, `dto/create-post.dto.ts`, `dto/update-post.dto.ts`, `dto/create-comment.dto.ts`.

- [ ] **Step 1: 요청 DTO `@ApiProperty` — enum 포함**

`create-post.dto.ts`의 `category`는 **enum 명명 스키마**로 표기(이번 요구의 핵심): `@ApiProperty({ enum: PostCategory, enumName: 'PostCategory', required: false, example: PostCategory.FREE })`. `enumName`을 주면 허용값(NOTICE|FREE)이 재사용 명명 스키마(`#/components/schemas/PostCategory`)로 분리된다. 기존 `@IsOptional()`/`@IsEnum(PostCategory)` 유지.
- `update-post.dto.ts`: title·content `@ApiProperty`.
- `create-comment.dto.ts`: content `@ApiProperty`.

- [ ] **Step 2: 컨트롤러 데코레이터** — `@ApiTags('board')`. 컨트롤러 레벨 `@UseGuards(JwtAuthGuard)`이므로 컨트롤러에 `@ApiBearerAuth(SWAGGER_BEARER_AUTH)` 1회. 라우트별 `@ApiOperation` + 성공 `@ApiResponse`(생성 201, DELETE 204) + 에러:
- `POST /buildings/:buildingId/posts`: 201, 403(멤버 아님).
- `GET /buildings/:buildingId/posts`: 200(배열), 403.
- `GET /posts/:postId`: 200, 403, 404.
- `PATCH /posts/:postId`: 200, 403(작성자 아님), 404.
- `DELETE /posts/:postId`: 204, 403, 404.
- `POST /posts/:postId/comments`: 201, 403, 404.

- [ ] **Step 3: 빌드 + 테스트 불변** — `npx tsc --noEmit && npx jest src/board` → PASS.

- [ ] **Step 4: Commit**

